一、基礎核心系統配置 (System Core)

專案採用模組化 Web 架構，無需下載 APP 即可執行，有效降低使用門檻並提升參與率。

狀態控管模組
State Management Module
  • 初始邏輯判別：系統自動偵測當前資料狀態，執行對應資源預載，確保遊玩體驗流暢。
  • 進度存取節點：採用瀏覽器端存儲技術，記錄當前進度與持有物件，防止重新整理後資料遺失。
系統主控模組
Dashboard Module
  • 環境運行檢測：自動判定裝置型號與瀏覽器版本，若環境不符將提示使用者，確保遊戲穩定運行。
  • 主功能控制列：包含「遊戲指南」、「文獻檔案」與「狀態設置」之控制介面。
互動解謎模組
Puzzle Module
  • 核心任務功能：進行互動操作與解謎行為的核心功能，如圖文線索、物件蒐集、答案比對運算等。
  • 即時反饋機制：包含驗證判定、錯誤反饋、卡關提示以及通關過場效果。

五、專案服務流程說明 (Project Workflow)

本項目旨在說明本專案之執行流程與各階段作業內容，以協助業主理解專案進行方式與節點安排。
實際權利義務、責任歸屬與相關限制，皆以正式合約條款為準。

01. 初步接洽與需求了解
專案開始前，將進行初步接洽與溝通，了解業主的專案目的、使用情境、預期規模與時程需求；並依據經驗說明可行之開發方式、整體流程、預估時程與方案方向，協助業主建立對專案全貌的理解。
階段目的 釐清方向與共識，作為後續評估與規劃之基礎。
02. 需求評估與場域預查
依據業主提供之場域相關資料（如照片、空間說明、網路環境概況、預期動線等），執行方將進行初步可行性評估，判斷是否適合進行實境解謎專案之設計與配置。
作業性質 此階段屬於前期書面評估，實際執行細節仍將於後續實地探勘時進一步確認。
03. 報價及合約簽訂
於需求方向確認後，將提供對應項目之正式報價單與專案合約書，說明所包含之功能範圍、服務項目、整體費用結構以及雙方之權利義務與約定事項，作為雙方後續合作與執行之依據。
04. 合約簽署與專案啟動
業主確認報價內容與規格無誤後，簽署正式專案合約，並支付報價的 50% 作為訂金。
啟動規範 訂金將用於前期開發使用，待款項入帳確認後，執行方於 3 個工作天內 啟動專案，並依流程展開後續執行。
05. 實地探勘與環境評估
專案啟動後，執行方將視約定安排實地探勘，確認實際場域條件、動線配置與設計可行性，並蒐集關卡設計與系統配置所需之環境資訊。
風險控管機制 如實地情況與前期資料存在重大落差，雙方將就後續執行方式進行溝通與調整或終止項目。
訂金視為專案啟動與前期規劃之勞務對價。若協議終止，訂金優先抵扣已履行之勞務成本，不予退還；剩餘款項（如有）再依已完成工作比例協議結算。
06. 素材設計與階段確認
設計與製作關卡配置、文案與圖像等素材。此階段完成後，將提供設計稿（含文案、圖像、謎題邏輯說明）請業主進行校稿。
校對規範 提供包含 2 次 階段性校稿修訂。業主需於收到校稿文件後 5 個工作天內 回覆確認或提出修改建議，若業主於期限內有突發狀況，可以書面告知執行方，經雙方協議後得延長校稿期程。
回覆時，將請業主於同一批文件中提出針對欲調整項目之修訂意見，此將視為一次校稿回合。
原則上，逾期未回覆將視同確認無誤，逕行進入程式開發階段。
07. 系統開發與功能實作
依據報價單上的規格項目以及校稿確認後的文件，進行系統開發與功能實作，包含關卡框架建構、加購模組及服務導入等工程整合。
變更聲明 本階段原則上僅以「已確認之設計稿」為實作基準。如需於此階段變更已確認之素材或邏輯，可能影響交付時程並需視變更程度另行評估成本。
08. 測試反饋與專案驗收
完成系統實作後，執行方將提供完整可執行版本，供業主進行測試與驗收，確認是否符合正式報價項目與專案合約之約定內容。
