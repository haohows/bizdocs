序、地方創生互惠合作說明

這份文件是給希望嘗試「地方 × 實境解謎 × 手機互動」的地方創生團隊參考。許多地方團隊本身已經累積了很好的故事、場域、人際連結與在地觀察，但不一定有足夠的人力與預算，從零開始建置一套可在 LINE 中運作的互動系統。

因此，本方案想做的事情很單純：由合作單位提供地方內容、文史資料、謎題想法、素材與現地規劃；由實境效應事務所負責 WEB in LINE 互動系統的設計與實作。雙方一起把原本存在於地方的故事與路線，整理成一個玩家可以實際走訪、閱讀、解謎與互動的作品。

這不是要取代紙本、LINE 或原本的地方導覽方式，而是在既有內容上增加一層可用手機操作的互動體驗。玩家可直接從 LINE 進入活動頁面，不需另外安裝 App；合作單位也可以延續既有 LINE 官方帳號、地方活動或店家串連作為入口。

若作品完成後雙方都認同，也希望這份成果能同時成為合作單位推廣地方的活動內容，以及我方後續介紹 WEB in LINE 服務時的實際案例。換句話說，這是一個以「共同完成作品」為核心的互惠合作：合作單位提供地方內容與現地經驗，我方提供系統設計與實作，雙方一起讓作品成形。

一、基礎核心系統配置

採用模組化 Web 架構設計，適用於各類型專案架構並加快開發速度。

模組名稱 狀態控管模組
  • 環境運行檢測：自動判定裝置與運行環境，若環境不符將予以提示，確保穩定運行。
  • 初始邏輯判別：系統自動偵測當前資料狀態，執行對應資源預載，確保遊玩體驗流暢。
模組名稱 系統主控模組
  • 進度存取節點：採用瀏覽器端存儲，記錄進度與持有物件，防止重新整理後資料遺失。
  • 主功能控制列：包含「遊戲指南」、「文獻檔案」與「狀態設置」之控制介面。
模組名稱 互動解謎模組
  • 核心任務功能：進行互動與解謎的核心功能，如圖文線索、物件蒐集、答案比對運算等。
  • 即時反饋機制：包含驗證判定、錯誤反饋、卡關提示以及通關過場效果。

項目名稱 劇情引導
用於場景描述、動線說明與行動指引的內容。
項目名稱 資訊文獻
以圖文方式呈現，用於補充事件背景與相關資料的說明內容。
項目名稱 關卡與謎題

關卡中供玩家進行互動的謎題、線索與行動指引等內容。

若需其他線索呈現模式（如：AR 擴增圖、鏡像貼圖等），可依作品需要討論是否納入本次合作範圍。

項目名稱 判定與驗證
檢核是否符合通關條件的判斷機制，基本配置為單組文字驗證(嚴謹模式)，需輸入完全相符的答案，包含大小寫、全半形與標點符號等。
若需其他驗證機制（如：圖像辨識），可依作品需要討論是否納入本次合作範圍。
項目名稱 LINE 提示入口串接

當回答錯誤時予以視覺回應，並顯示提示 (Hint) 按鈕，供玩家自行決定是否查看。

若點擊提示按鈕，可開啟 LINE OA 對話畫面；LINE OA 回覆內容與互動流程依業主既有設定為準。

2. 可延伸的系統功能

項目類別 客製化驗證開發
依需求開發特殊解謎機制，如：邏輯排序、拖曳解謎、小遊戲通關挑戰等非標準模組。
項目類別 金鑰發行與驗證模組
採用數學演算法 進行本機驗證，可判斷玩家持有金鑰之有效性。
  • 合作單位可自行於電腦上無限量產生成金鑰。
  • 因採離線驗證機制，金鑰具可重複使用特性，若需嚴格注意事項「單一金鑰僅能使用一次」，請參考下方的雲端授權平台。
  • 合作單位亦可採用虛實整合的方式，將部分解謎道具實體化（如謎題包），即便金鑰被分享，玩家若無實體道具仍無法遊玩。
  • 本方案僅提供軟體端之驗證機制，實體道具之設計與製作需由合作單位自行規劃。
項目類別 雲端序號授權管理平台
提供一套標準化資料庫結構與公版序號管理後台，用於產生新序號、管理既有遊戲序號，以及後台本身的基本權限控管。
  • 後台設有帳號管理機制，以避免未授權人員存取與操作，遊戲端將加裝序號驗證機制，序號使用後即刻失效，杜絕重複使用。
  • 本系統以既有標準功能架構交付，適用於單一專案之序號產生與管理需求，若需要調整資料結構或擴充後台功能，會再一起確認可行做法。
  • 外部帳號與部署提醒：系統交付時，會請合作單位依指引，申請雲端資料庫以及部署空間；相關帳號由合作單位持有與管理，後續流量與伺服器使用狀況，需由合作單位自行管理。

五、合作流程說明

互惠合作的重點，是先把地方內容與系統實作慢慢對齊，讓作品能從想法變成可實際遊玩的體驗。以下流程用來協助雙方理解合作會怎麼進行，實際細節可依專案狀況調整。

01. 先聊地方題材與作品方向
先了解合作單位想呈現的地方故事、街區場域、活動目的、玩家對象與目前已有的素材。這個階段的重點，是判斷這個題材是否適合轉成 WEB in LINE 的周遊型互動體驗。
02. 確認共同投入與合作範圍
雙方一起確認本次作品大約會做到什麼程度，例如關卡數量、互動形式、是否需要 LINE／LIFF 串接、是否需要進階模組，以及各自負責的工作內容。
若會使用雲端服務、第三方工具、印刷、場地或其他非系統開發資源，會在這個階段先確認準備方式與管理方式。
03. 整理素材、謎題與動線內容
合作單位提供地方資料、照片圖文、謎題邏輯、答案、提示內容、現地動線與必要的場域資訊；我方則協助確認這些內容是否足以轉成系統畫面與互動流程。
04. 系統開發與互動流程實作
內容方向確認後，我方依照已整理的資料，進行 WEB 互動系統、關卡框架、答題驗證、進度保存、LINE／LIFF 入口串接與約定互動模組的實作。
05. 測試、回饋與調整
完成測試版本後，提供合作單位試玩與確認。若畫面內容、流程、答案驗證或操作體驗需要調整，會依已確認的合作範圍進行修正。
06. 上線、交付與活動使用
確認作品可正常遊玩後，依雙方討論的方式完成部署、交付或上線。合作單位可將作品用於地方活動、導覽體驗、成果展示或後續推廣。
07. 案例整理與後續延伸
作品完成後，雙方可再確認成果介紹方式，例如可否使用作品名稱、畫面截圖、合作單位名稱、活動照片或案例文字。若後續想增加關卡、擴充功能或延長營運，也可以再一起討論後續方向。

六、系統環境與注意事項

  • 裝置注意事項：
    本專案不支援電腦、平板及 iPhone X、iPhone 8 系列及更舊機種。
  • 建議配備：
    建議使用 iPhone 11 (或 2021 年後 Android 機型) ，且螢幕尺寸大於 5.5 吋以上之手機遊玩。
  • 軟體環境：
    為確保遊戲體驗，建議將作業系統與瀏覽器更新至最新版本，並限定使用 LINE 應用程式開啟。
  • 畫面呈現：
    本專案為 Web 形式之互動遊戲，實際畫面呈現將依不同手機型號、螢幕比例有所差異，屬正常顯示情形。

七、交付形式與製作工期

  • 交付形式：
    本專案交付成果與部署方式依雙方合作文件或合約約定辦理；若需交付編譯後之專案靜態壓縮檔，亦可供合作單位保存備份。常見部署方式可分為「標準託管部署」與「合作單位環境部署」兩類型：
    • 標準託管部署：
      專案之 LIFF 入口與相關設定建置於我方管理之 LINE Developers 環境，並部署於我方管理之 Cloudflare 帳號。託管期間與後續維持方式，依雙方合作共識辦理。
      託管期滿後，雙方再確認是否繼續維持線上版本；若不再維持，線上版本將下架。
    • 合作單位環境部署：
      專案之 LIFF 入口與相關設定建置於合作單位提供之 LINE Developers 環境，並部署於合作單位提供之 Cloudflare 帳號，交付完成後由合作單位自行持有與管理。
      後續若因合作單位自行更改 LINE Developers、LIFF、Cloudflare、DNS、網域或相關設定導致無法連線，可再依狀況協助釐清或由合作單位自行處理。
    • 正式環境可透過 LINE App 正常開啟後，即視為交付完成。
    • 靜態壓縮檔供保存備份與後續重新部署使用；若需重新上線，仍須搭配 Cloudflare、LINE Developers／LIFF 等相關環境設定。
  • 製作工期：
    標準導入專案製作週期約為 3～6 週，此時程特指「系統開發與功能實作」之部分。
    若因合作單位延遲完成素材上傳，或作品內容需要增加客製開發，交付日期將視實際狀況調整。
